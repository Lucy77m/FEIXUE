## 1. Mochi 是什么

Mochi 同时是两样东西：

- 🐾 **一只有生活的桌宠** —— 纯代码绘制（没有任何贴图素材），会眨眼、跟着鼠标看、发呆哼歌、钓鱼喝咖啡、丢球接球；机器发烫会扇风、下雨会撑伞、垃圾堆多了抓虫、拖文件能喂它、纪念日端蛋糕；你不理它会自己找乐子，你离开它会打瞌睡，偶尔还会主动找你说句话。
- 🧠 **一个能操控整台电脑的本地 Agent** —— 接你自己的大模型（任意 OpenAI 兼容接口），能看屏幕、点窗口、动鼠标键盘、跑命令、写代码、读写文件、上网搜索、记东西、查资料；还能**定时盯屏、改完代码自己跑测试、并行编排一群分身、按每天/每周重复提醒**……把"和 AI 对话"变成"让 AI 替你动手"。

它有持久的**情绪与亲密度**，会随相处慢慢长出一份**自我画像（人格演化）**，所以是"同一只"，不是每次重置的聊天框。

> 它不是一个聊天框，是一个**活在你电脑里、有身体也有能力的存在**。

## ✦ 设计哲学：一个全权的伙伴

Mochi 在你的机器上**和你本人同等权限**地行动 —— 跑任意命令和代码、动鼠标键盘、读写文件。这是**刻意的**：

- **不做沙箱、不层层设限。** 把它关进笼子，就废掉了"它能真的替你动手"这个前提。我想要的是一个全权的伙伴，不是一个处处要批准的受限助手。
- **权力在你手里，但默认放开。** 控制面板能按组关掉能力（联网 / 操控 / 命令），`confirm` 面板会在不可逆操作前拦一道 —— 这些是给你**需要时**用的闸，不是默认套在它脖子上的链子。
- **安全护栏只挡"灾难级手滑"。** `executor/safety.py` 只硬拦极小一撮不可逆的毁灭操作（格式化磁盘、删系统根等），并不试图防住一个成心使坏或被注入的模型 —— 那不是这层能解的，也不是它的职责。它是接住模型偶尔抽风的网，不是牢笼。
- **所以这是个信任你、也请你同等回以判断的工具。** 它有多大能耐，你就担多大责任；在你信得过的机器上，把它当伙伴用。

> 一句话：我宁可它**能力过剩、需要你把着方向盘**，也不要它**安全到什么都干不成**。

### 3.2 设计原则

- **UI 在主线程，agent 在工作线程**，跨线程一律走 Qt 信号（queued）。
- **动画是时间的连续函数**：一切绑定全局时间戳 `self._t`，不做逐帧随机抖动 —— 确定性、可重放、不"神经质"。
- **行为数据驱动**：情绪标签、动作、服装集中在表里，新增一项＝加一行 + 一个画法/曲线，而非又一个 `if/elif`。
- **show, don't tell**：模型只拿到"心情氛围"，从不拿到也不报数字。
- **降级链无处不在**：嵌入没有→子串检索；模型失败→提醒仍念原文；任何工具异常→变成可读文本回灌，绝不中断循环。
- **克制**：所有"自己动起来"的行为都裹多重门控（冷却、上限、亲密度、在场）。

### 3.3 线程模型

```text
主/UI 线程 (Qt event loop)
  └─ PetWindow(60fps) · 气泡/输入框/黑板/拍立得/思考气泡/确认面板 · 托盘 · QTimer
        │ Signal(queued)
        ▼
工作线程 (QThread)  ──  AgentWorker → Agent.run()  [阻塞：网络 + 工具]
        │ 派生 daemon
        ▼
  子代理(并发≤4) · 后台任务(≤3) · 反思(≤1) · MCP 事件循环 · 热键消息循环
```

权威 busy 判定用 worker 的 `is_running`（单 bool 跨线程读安全），不用滞后的 UI 状态。每个 agent 有独立的 PowerShell + Python 子进程；网络 / 只读文件 / 各自 shell·python 可并发，共享资源（鼠标 / 屏幕 / 记忆 / pip）走调度锁串行。

### 3.4 Agent 循环

`用户消息 → 模型 → 工具调用 → 执行 → 回灌 → 重复 → 回复`，关键设定：

| 参数 | 值 | 含义 |
| --- | --- | --- |
| 单回合工具步数上限 | 16 | 防止无限自我调用 |
| 子代理深度 | 1 | 宠物可派分身，分身不能再派 / 丢后台（防失控递归） |
| 并行子代理 | 4 | 同时扇出上限 |
| 历史 token 预算 | 24000 | 按估算 token 裁剪（非消息条数），且 tool-call-aware |
| 单条工具结果 | 8000 字 | 入历史前截断 |
| 请求 / 尽力调用超时 | 120s / 45s | 用户回合 / 反思等后台调用 |

**关键机制**：

- **流式 + 思维链**：`streaming.py` 把 delta 流折回单条消息，逐片把 `reasoning_content` / `content` 喂给思考气泡；每个 chunk 检查取消，可在一句话中途停掉网络流。
- **打断**：点宠物即置 flag + 杀子进程（卡在长命令里也能立即解锁）；回合用历史标记回滚、不留痕；**打断不算失败**，不播沮丧动画。
- **截图与多模态成本**：只保留最新一张截图入历史，旧的换占位符；JPEG 最长边缩到 ≤1600px，但坐标始终报真实分辨率。
- **后台长任务**：长活儿用 `start_background_task` 丢后台异步跑（信号量 = 3），主线程继续陪你聊，办完汇报 —— "边跑边聊"；运行中可 `list_background_tasks` 查看、`stop_background_task` 协作式叫停。
- **确定性子代理编排**：`spawn_workflow` 一次把多个子任务**并行扇出**（≤4 并发）或串成**流水线**（上一阶段输出注入下一阶段），还能要求子代理按指定 JSON 形状结构化返回 —— 把"靠模型恰好同轮多次派分身"的偶发并行变成可靠原语；子代理 / 后台代理按深度被剔除编排与看屏工具，防递归失控与越权。
- **反思与人格演化**：实质性回合结束后起一个反思调用（信号量 = 1，忙则跳过），蒸馏出经验 / 偏好 / 环境事实 / 情景日志，并**缓慢重写自我画像**（见 3.6）；纯短闲聊或只用了装饰性工具的回合跳过反思。

### 3.5 情绪状态机

连续的 valence / arousal 心情 + 缓慢累积的 rapport（亲密度），持久化到 `data/emotion.json`，按真实流逝时间衰减 —— 所以是"同一只"。

- **实时衰减**：心情每次读取都从上次事件锚点衰减到当下，与轮询频率无关，引擎用锁守护跨线程并发安全。
- **Appraisal 事件**：每条用户消息、任务成 / 败、启动归来、夸 / 骂等都按权重改动心情；夸 / 骂由本地启发式扫描用户消息命中（保守匹配，剔除否定句与指向用户自己的假命中）。
- **rapport「难涨慢落」**：越熟涨得越慢、渐近 1.0；负向直接扣；有 0.15 地板，长期冷落只能侵蚀到地板，"发生过的羁绊会留痕"。
- **心情→状态**：映射到 `excited / content / anxious / down` 等标签，只染色语气，从不报数字。
- **行为选择**：按心情加权随机选动作（高斯 VA 亲和度 × 稀有度 × 最近性 × 亲密度），而非 `if tag=="happy"` 硬编码。

### 3.6 人格演化（自我画像）

`persona.py` 维护一份独立于设定的"演化层"，存于 `data/persona.json`：

- 出厂底色（种子）写在 `prompts.py` 里；persona.json 存的是**它跟你相处下来长出来的那一层**，空 = 还没长出个性、纯底色。
- 每轮反思**重写而非堆积**（上限 600 字）—— 画像始终是"当下的自己"，不滚雪球，且有惯性（在旧画像上缓慢演化）。
- 通过 `as_context()` 以"【你是谁】"前缀注入每个回合的上下文，自然成为说话做事的底子。
- "清空记忆"会一并清空它，回到出厂底色。

### 3.7 确认面板

`pet/confirm.py` 的 `ConfirmBox` 是浮在宠物身边的"执行 / 不执行"小面板；agent 通过 `confirm` 工具在**不可逆 / 高风险操作前**（删文件、覆盖重要文件、`git push --force`、清数据、关机等）弹出并**阻塞等待用户点击**，也可用于主动提议某个改动让你拍板。面板跟随宠物移动，点了才返回结果继续。

### 3.8 其它子系统

- **眼 / 手 / 截图**：截图用 `SetWindowDisplayAffinity` 把宠物窗口标记为"对用户可见、对截图隐形"；优先用 UIAutomation 无障碍树读控件名 + 精确坐标直接点，读不到才退回截图找图；中文输入走剪贴板 + Ctrl+V。
- **在场感知**：用 Win32 全局最后输入时间判断你在不在，久无输入打瞌睡（深夜阈值更短），一动就醒。
- **主动消息**：`proactive.py` 管冷却 / 每日上限分档（安静 / 正常 / 话痨），`app.py` 每 60s 轮询、全部门控（不忙 / 在场 / rapport 达标 / 冷却已过）通过才开口；回来问候有最小间隔，绝不边聊边插。
- **记忆 / 知识库 / 情景日志**：三套独立持久化 —— 记忆是"它学到的关于你的"，知识库是"你喂的外部文档（RAG）"，情景日志是"最近做了啥"，严格分离。
- **提醒 / 定时**：`say`（到点用自己口吻说）/ `do`（到点后台真去办并汇报），支持**每天 / 每周 / 每隔 X 分钟**重复（持久跨重启，关机错过只补最近一次、不刷屏）；可 `list_reminders` 列出、`cancel_reminder` 撤销；MoChi 隐藏或被全屏游戏挡住时改走**系统托盘通知**触达。一律走调度器持久化，绝不让模型自己 sleep 等时间。
- **定时看屏**：`watcher.py` —— 按你设的间隔截当前窗口图、就你给的关注点分析并播报（如盯游戏局势、有危险/机会就提醒）；会话级（不落盘、重启即止），结果回来时复检状态，已关机 / 已停 / 正对话中就不打扰，截图瞬时失败也不白烧一个周期。
- **工程纪律**：`executor/devtools.py` 的 `review_diff`（看未提交 diff，可限定文件/子目录）/ `run_tests`（自动探测 pytest · npm，独立 5 分钟超时、超时杀整棵进程树）；系统提示有一节"动代码库时"的纪律 —— 改前看现状、小步外科手术、**改完跑测试 / 自查 diff**、默认分支先开分支、不可逆 git 先 confirm。
- **隐身 / 入场 / 穿越**：拖到屏幕边缘缩成小角偶尔探头；每次启动随机一种入场动画且绝不重复上次；偶尔"虫洞穿越"——原地裂开虫洞、旋转缩入、在窗口不可见时瞬移、从屏幕另一处冒出来。
- **节日 / 陪伴感**：`occasions.py` 认公历节日 + 你填的生日，到日子时给模型一句应景"由头"，让它自然提起而非干巴巴道贺；`stats.py` 默默记着初次相遇时间与累计互动次数，是它"认识你多久了"的底气。
- **伴生行为（companions/）**：五台各管一摊的"小机器"，都裹在场 / 忙 / 亲密度 / 冷却门控里——
  - `sensors.py`：十秒一次读 CPU / 内存 / 电量体征（带迟滞，不抖），驱动**机器拟态**（高烧扇风、内存压扁、低电量、深夜盖被、冬天蹭暖）；查麦克风占用做**会议静音**；两小时一次 `wttr.in` 查天气做**天气拟态**；盯下载目录与桌面图标数；瞄焦点是不是密码框就**捂眼**。
  - `playtime.py`：玩耍与物理反馈——丢球接球、窗台蹲守（窗口一动就摔）、挠痒 / 拖甩记仇、走动留**墨水脚印**、钓鱼有渔获彩蛋；扫 temp 超 500MB 生**垃圾虫**，点死即真清理。
  - `rituals.py`：**仪式感**——晨间心情预报、纪念日蛋糕（吹蜡烛）、退出挥手道晚安、番茄钟 25 分钟专注。
  - `feeding_ctrl.py` + `pet/feeding.py`：**投喂分流**——拖进来的文件按类型走（垃圾→回收站、文档→知识库、图片→直接看），受保护 / 危险路径拦下不吃，大餐 / 整目录先弹确认。
  - `watchers.py`：**后台守望**——盯 `start_background_task` 起的后台 shell，成功庆祝、失败叫 agent 分析；把剪贴板小宝贝悄悄收着，隔几小时亲昵"回赠"。
- **身体感受（somatic.py）**：被喂 / 被摔 / 接到球 / 蛋糕端出来这些事，连同"在开会""机器高烧"等持续状态，统一汇成一段"身体近况"注入每轮上下文——所以它聊天时**真的知道刚才身上发生了什么**，而不是凭空装。
- **MCP / 热键 / 技能 / 审计 / i18n**：MCP 连接器以 `mcp__{server}__{tool}` 融入工具表；全局热键独立线程跑 Win32 消息循环（唤出 / 问选区 / 顺手改写）；技能把跑通的代码存成可复用项注入提示；所有工具调用记审计日志；控制面板 UI 支持中 / 英 / 日。
