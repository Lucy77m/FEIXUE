## 1. Mochi とは

Mochi は同時に二つのものです。

- 🐾 **生活を営むデスクトップペット** —— 完全にコードで描画（一切の画像素材なし）。まばたきし、マウスを目で追い、ぼんやり鼻歌を歌い、釣りをしコーヒーを飲みます。構わずにいると自分で遊びを見つけ、あなたが離れると居眠りし、ときには自分から話しかけてきます。
- 🧠 **PC 一台をまるごと操作できるローカル Agent** —— あなた自身の大規模言語モデル（任意の OpenAI 互換インターフェース）に接続し、画面を見て、ウィンドウをクリックし、マウスとキーボードを動かし、コマンドを実行し、コードを書き、ファイルを読み書きし、ネットを検索し、物事を記憶し、資料を調べます……「AI と会話する」を「AI に自分の代わりに動いてもらう」へと変えます。

永続的な**感情と親密度**を持ち、付き合いを重ねるうちに少しずつ**自己像（人格の進化）**を育てていきます。だからこれは「同じ一匹」であり、毎回リセットされるチャット欄ではありません。

> これはチャット欄ではなく、**あなたの PC の中で生き、身体も能力も持つ存在**です。

## 2. 機能ハイライト

### 🧠 仕事ができる頭脳

| 能力 | 説明 |
| --- | --- |
| **コマンド & コード** | PowerShell / cmd、永続的な Python 環境（pip でライブラリ導入、API 呼び出し、自動化の駆動が可能） |
| **ファイル & コードベース** | ファイルの読み / 書き / 精密編集、正規表現でのコード検索、名前でのファイル探索 |
| **ネット接続** | Web 検索、本文抽出、HTTP リクエスト、パッケージ導入 |
| **画面の閲覧 & 操作** | スクリーンショット、OCR による文字認識（RapidOCR）、画面内の画像検索、アクセシビリティツリーでコントロールを読み取って精密クリック、マウス・キーボード |
| **記憶** | 長期記憶（経験 / 嗜好 / 環境）+ エピソードログ + ナレッジベース（ドキュメント RAG）。対話のたびに自動で振り返り定着させる |
| **スキル** | 検証済みのやり方を再利用可能なスキルとして保存し、次回そのまま呼び出す（Voyager 式「使うほど強くなる」） |
| **拡張** | MCP コネクタ、分身を派遣してサブタスクをこなす、長時間タスクをバックグラウンドに投げて非同期実行（走らせながら会話） |
| **確認ガードレール** | 不可逆 / 高リスクな操作の前に「実行 / 中止」パネルをポップアップし、クリックして初めて先へ進む |
| **スケジュール** | 時刻になったらリマインド（Mochi 自身の口調で伝える）、定刻に自動で用事をこなす |

### 🐾 ぬくもりのある寄り添い

| 観点 | 説明 |
| --- | --- |
| **感情システム** | valence / arousal / 親密度が表情と行動を駆動。仲が深まるほど打ち解ける。褒めれば喜び、けなせば落ち込む |
| **人格の進化** | あなたと付き合う中で、振り返りのたびに「自己像」を少しずつ書き直し、話し方や振る舞いの土台に注入する —— 関係の中で育つ「自分」 |
| **自発的アニメーション** | 15 種の思考ポーズ、ぼんやり吹き出し、マウスを追う視線。すべて時間の連続関数で、ガタつかない |
| **小道具つきの小芝居** | コーヒーを飲む、釣りをする、事件を調べる、本を読む、音楽を聴く、ゲームをする、星を眺める —— どれも段階に分かれた小さな劇 |
| **単発アクション** | ダンス、歓声、くるくる回転…… それぞれに合わせた特効つき（紙吹雪 / 音符 / 残像） |
| **在席感知** | あなたが離れれば居眠りし、戻れば目覚める。画面の端へドラッグすると「隠れ」、小さな角だけ覗かせてこっそり見守る |
| **自発的な発話** | 暇なときにたまに自分から口を開く。しかも**節度をもって** —— 長いクールダウン、1 日の上限、親密度のゲート制御で、決して連投しない |
| **構造化された表現** | 比較 / リスト / コードはそばの小さな黒板に描いて解説。画像 / GIF の表示も可能 |

### ⌨️ 手になじむインタラクション

- **グローバルホットキー**：`Ctrl + Alt + S` でどこからでも入力欄を呼び出し。`Ctrl + Alt + A` で選択したテキストをそのまま質問
- **コントロールパネル**：インターフェース / モデルパラメータ / 返答言語 / 能力スイッチ / 自発頻度（静か · 普通 · おしゃべり）/ ワンクリックで「記憶を消去し、新生児のように」設定

## 3. インストールと実行

**環境**：Windows 10 / 11 · Python ≥ 3.11 · [uv](https://github.com/astral-sh/uv)

```powershell
# 依存関係をインストール
uv sync

# 実行
uv run python main.py
```

初回起動時にコントロールパネルが表示されるので、お使いの大規模言語モデルの設定を入力してください。

- **API Key** / **インターフェースアドレス（base_url）** —— 任意の **OpenAI 互換**サービス（通義千問 / DeepSeek / OpenAI / ローカル vLLM、Ollama……）
- **対話モデル** —— メインのやり取りに使う chat モデル
- **埋め込みモデル** —— ナレッジベースと記憶の意味検索に使用（埋め込みエンドポイントがなければ自動で部分文字列検索にフォールバックするため、未入力でも可）
- **返答言語 / 能力スイッチ / 自発頻度 / 発散度と最大長 / 思考過程を表示するか** —— いずれもパネル内で調整でき、各ターンでリアルタイムに反映される

設定が済むと、Mochi は画面の隅から「入場」してきます。クリック（または `Ctrl + Alt + S`）で文字を打ってチャット。タスクトレイのアイコンからいつでもパネルを再表示できます。

> 設定はローカルの `data/settings.json` にのみ保存され、どこにもアップロードされません。データディレクトリは環境変数 `STAR_DATA_DIR` で上書きできます。

## 4. アーキテクチャ設計

### 4.1 ディレクトリとモジュールの責務

```text
desktop_pet/
├─ app.py            # 総司令塔：UI / agent / タイマー / トレイ / ホットキーを接続、自発メッセージのリアルタイムゲート制御
├─ agent/            # 頭脳
│   ├─ loop.py       #   agent ループ：モデル↔ツールのフィードバック、ストリーミング、トリミング、振り返り、人格進化
│   ├─ tools.py      #   ツール表（約 60 個）とディスパッチ、並行安全ロック
│   ├─ streaming.py  #   delta ストリームを 1 件のメッセージに畳み込み、思考過程の分割
│   ├─ prompts.py    #   すべてのプロンプト（人格シード、システム、振り返り）を一箇所に集約
│   └─ progress.py   #   思考ポーズのスケジューリングと進捗表示
├─ pet/              # 身体：ウィンドウ、コード描画のキャラクター、発話/入力、黒板、コントロールパネル、確認パネル、
│                    #       隠れ、入場、行動セレクタとアクションライブラリ、小道具とパレット
├─ emotion/          # 感情ステートマシン（VA + 親密度）と感情タグ表
├─ persona.py        # 自己像の進化レイヤー（persona.json）、対話コンテキストへ注入
├─ memory/           # 長期記憶（SQLite）+ ベクトル埋め込み
├─ executor/         # コマンド / Python / ファイル / ネットワーク / 視覚（OCR·画像検索）/ システムメモリ / 安全ガードレール
├─ hands/            # マウス / キーボード / ウィンドウ制御
├─ eyes/             # スクリーンショット + アクセシビリティツリー（UIA）+ 画面内画像検索
├─ docs.py · reminders.py · proactive.py · journal.py · presence.py
└─ hotkeys.py · skills.py · mcp_hub.py · settings.py · audit.py · i18n.py
```

### 4.2 設計原則

- **UI はメインスレッド、agent はワーカースレッド**で動き、スレッド間は一律 Qt シグナル（queued）を経由する。
- **アニメーションは時間の連続関数**：すべてグローバルタイムスタンプ `self._t` に紐づけ、フレームごとのランダムなガタつきはしない —— 決定論的で、再現可能で、「神経質」にならない。
- **行動はデータ駆動**：感情タグ、アクション、衣装はすべて表に集約され、項目の追加＝1 行 + 1 つの描画法/曲線を足すだけ。また一つ `if/elif` を増やすのではない。
- **show, don't tell**：モデルが受け取るのは「気分の雰囲気」だけで、数値を受け取ることも報告することも決してない。
- **フォールバックチェーンが至る所に**：埋め込みがない→部分文字列検索；モデル失敗→リマインドは元の文面のまま読み上げる；ツールの例外はすべて読めるテキストに変えてフィードバックし、ループを決して中断しない。
- **節制**：「自分から動き出す」すべての行動は、複数のゲート（クールダウン、上限、親密度、在席）で包まれている。

### 4.3 スレッドモデル

```text
メイン/UI スレッド (Qt event loop)
  └─ PetWindow(60fps) · 吹き出し/入力欄/黒板/ポラロイド/思考吹き出し/確認パネル · トレイ · QTimer
        │ Signal(queued)
        ▼
ワーカースレッド (QThread)  ──  AgentWorker → Agent.run()  [ブロッキング：ネットワーク + ツール]
        │ daemon を派生
        ▼
  サブエージェント(並行≤4) · バックグラウンドタスク(≤3) · 振り返り(≤1) · MCP イベントループ · ホットキーメッセージループ
```

権威ある busy 判定にはワーカーの `is_running`（単一 bool でスレッド間の読み取りが安全）を用い、遅延のある UI 状態は使わない。各 agent は独立した PowerShell + Python サブプロセスを持ち、ネットワーク / 読み取り専用ファイル / 各自の shell·python は並行可能、共有リソース（マウス / 画面 / 記憶 / pip）はスケジューリングロックを経て直列化される。

### 4.4 Agent ループ

`ユーザーメッセージ → モデル → ツール呼び出し → 実行 → フィードバック → 繰り返し → 返答`。主要な設定値は以下のとおり。

| パラメータ | 値 | 意味 |
| --- | --- | --- |
| 1 ターンのツールステップ上限 | 16 | 無限の自己呼び出しを防ぐ |
| サブエージェントの深さ | 1 | ペットは分身を派遣できるが、分身は再派遣 / バックグラウンド投入不可（暴走的な再帰の防止） |
| 並列サブエージェント | 4 | 同時ファンアウトの上限 |
| 履歴 token 予算 | 24000 | 推定 token で（メッセージ件数でなく）トリミングし、かつ tool-call-aware |
| 1 件のツール結果 | 8000 字 | 履歴へ入れる前に切り詰め |
| リクエスト / ベストエフォート呼び出しタイムアウト | 120s / 45s | ユーザーターン / 振り返り等のバックグラウンド呼び出し |

**主要な仕組み**：

- **ストリーミング + 思考過程**：`streaming.py` が delta ストリームを 1 件のメッセージに畳み込み、`reasoning_content` / `content` を断片ごとに思考吹き出しへ供給する。各 chunk でキャンセルを確認し、一文の途中でもネットワークストリームを止められる。
- **割り込み**：ペットをクリックすると即座にフラグを立て、サブプロセスを kill（長いコマンドで詰まっていても即座にロック解除）。ターンは履歴マーカーでロールバックし、痕跡を残さない。**割り込みは失敗扱いにせず**、落胆アニメーションは再生しない。
- **スクリーンショットとマルチモーダルのコスト**：履歴には最新のスクリーンショット 1 枚のみ残し、古いものはプレースホルダに置き換える。JPEG は最長辺を ≤1600px に縮小するが、座標は常に実解像度で報告する。
- **バックグラウンドの長時間タスク**：長い仕事は `start_background_task` でバックグラウンドへ投げて非同期実行（セマフォ = 3）。メインスレッドは引き続きあなたと会話を続け、完了したら報告する —— 「走らせながら会話」。
- **振り返りと人格進化**：実質的なターンが終わると振り返り呼び出しを 1 つ起こし（セマフォ = 1、ビジー時はスキップ）、経験 / 嗜好 / 環境事実 / エピソードログを蒸留し、**自己像を緩やかに書き直す**（4.6 参照）。純粋な短い雑談や装飾的なツールしか使わなかったターンは振り返りをスキップする。

### 4.5 感情ステートマシン

連続的な valence / arousal の気分 + ゆっくり蓄積する rapport（親密度）を `data/emotion.json` に永続化し、実際の経過時間に従って減衰させる —— だから「同じ一匹」になる。

- **リアルタイム減衰**：気分は読み取るたびに前回のイベントアンカーから現在まで減衰し、ポーリング頻度に依存しない。エンジンはロックでスレッド間の並行安全を守る。
- **Appraisal イベント**：各ユーザーメッセージ、タスクの成功 / 失敗、起動時の帰還、褒め / けなしなどがそれぞれ重みづけで気分を変える。褒め / けなしはローカルのヒューリスティックでユーザーメッセージをスキャンして検出する（保守的なマッチングで、否定文やユーザー自身を指す誤ヒットを除外）。
- **rapport「上がりにくく下がりにくい」**：仲が深まるほど上がり方が緩やかになり、1.0 に漸近する。負の事象は直接差し引く。0.15 の床があり、長期の放置でも床までしか侵食できず、「起きた絆は痕跡を残す」。
- **気分→状態**：`excited / content / anxious / down` などのタグへマッピングし、口調を染めるだけで数値は決して報告しない。
- **行動選択**：気分による重みづけ乱択でアクションを選ぶ（ガウシアン VA 親和度 × レア度 × 最近性 × 親密度）。`if tag=="happy"` のようなハードコードではない。

### 4.6 人格進化（自己像）

`persona.py` は設定とは独立した「進化レイヤー」を維持し、`data/persona.json` に保存する。

- 出荷時の地色（シード）は `prompts.py` に書かれている。persona.json が保存するのは**あなたと付き合う中で育った、その一層**であり、空 = まだ個性が育っておらず純粋な地色のまま。
- 各ターンの振り返りで**積み増しでなく書き直す**（上限 600 字）—— 自己像は常に「今の自分」であり、雪だるま式に膨らまず、慣性を持つ（古い自己像の上で緩やかに進化する）。
- `as_context()` を通じて「【あなたは誰か】」という接頭辞で各ターンのコンテキストに注入され、自然と話し方や振る舞いの土台になる。
- 「記憶を消去」するとこれも一緒に消え、出荷時の地色へ戻る。

### 4.7 確認パネル

`pet/confirm.py` の `ConfirmBox` は、ペットのそばに浮かぶ「実行 / 中止」の小さなパネルです。agent は `confirm` ツールを通じて、**不可逆 / 高リスクな操作の前**（ファイル削除、重要ファイルの上書き、`git push --force`、データの消去、シャットダウンなど）にこれをポップアップし、**ユーザーのクリックをブロックして待ちます**。また、ある変更を自分から提案してあなたに判断を仰ぐのにも使えます。パネルはペットに追従して移動し、クリックして初めて結果を返して続行します。

### 4.8 その他のサブシステム

- **目 / 手 / スクリーンショット**：スクリーンショットでは `SetWindowDisplayAffinity` でペットのウィンドウを「ユーザーには見え、スクリーンショットには映らない」と標識する。優先して UIAutomation のアクセシビリティツリーでコントロール名 + 正確な座標を読んで直接クリックし、読めない場合のみスクリーンショットの画像検索へフォールバックする。中国語入力はクリップボード + Ctrl+V を経由する。
- **在席感知**：Win32 のグローバルな最終入力時刻でいるかどうかを判断し、入力が長くないと居眠りし（深夜は閾値が短くなる）、動けば目覚める。
- **自発メッセージ**：`proactive.py` がクールダウン / 1 日の上限のランク分け（静か / 普通 / おしゃべり）を管理し、`app.py` が 60s ごとにポーリングして、すべてのゲート（ビジーでない / 在席 / rapport 到達 / クールダウン経過）を通過したときだけ口を開く。帰還時の挨拶には最小間隔があり、会話中に割り込むことは決してない。
- **記憶 / ナレッジベース / エピソードログ**：3 つの独立した永続化 —— 記憶は「Mochi があなたについて学んだこと」、ナレッジベースは「あなたが与えた外部ドキュメント（RAG）」、エピソードログは「最近何をしたか」で、厳密に分離されている。
- **リマインド / スケジュール**：`say`（時刻になったら自分の口調で言う）/ `do`（時刻になったらバックグラウンドで実際にこなして報告する）。いずれもスケジューラを経て永続化し、モデルに自分で sleep して時間を待たせることは決してしない。
- **隠れ / 入場**：画面の端へドラッグすると小さな角に縮んでときどき顔を覗かせる。起動のたびにランダムに 1 種類の入場アニメーションを選び、前回と決して重複させない。
- **MCP / ホットキー / スキル / 監査 / i18n**：MCP コネクタは `mcp__{server}__{tool}` としてツール表に溶け込む。グローバルホットキーは独立スレッドで Win32 メッセージループを回す。スキルは動作確認済みのコードを再利用可能な項目として保存しプロンプトへ注入する。すべてのツール呼び出しは監査ログに記録する。コントロールパネル UI は中 / 英 / 日に対応。

> 📐 内部の仕組み（完全なステートマシン / ゲート / 実際のパラメータ）を深く知りたい → **[設計ドキュメント DESIGN.md](DESIGN.md)**

## 5. パッケージ配布（Windows · PyInstaller）

Mochi を Python 環境に依存しない、ダブルクリックで使える Windows プログラムにパッケージングします。

### 5.1 ワンクリックパッケージ

```powershell
.\build.ps1
```

これは手動で以下を実行するのと等価です。

```powershell
uv sync                                   # 依存関係をインストール（dev グループの pyinstaller を含む）
uv run pyinstaller mochi.spec --noconfirm
```

成果物：**`dist\Mochi\Mochi.exe`** —— 配布の際は**`dist\Mochi\` ディレクトリ全体**を一緒に相手へコピーしてください（exe の隣にあるものの山が、その依存関係です）。

### 5.2 主要な設計

| 項目 | 選択 | 理由 |
| --- | --- | --- |
| 形態 | **onedir**（ディレクトリ版） | 起動が速い。RapidOCR のモデルは比較的大きく、単一ファイル版は毎回の解凍が非常に遅くなる |
| コンソール | **なし**（`console=False`） | ダブルクリックで黒い窓が出ない |
| データディレクトリ | **`%APPDATA%\Mochi`** | パッケージ後の exe の隣は読み取り専用 / 書き込み権限なし。設定、記憶、ログはすべてここに書く |

開発時（`uv run python main.py`）は引き続きプロジェクト内の `data\` を使い、パッケージ版は `%APPDATA%\Mochi` を使う。どちらも `STAR_DATA_DIR` で上書き可能。

### 5.3 独立した Python を内蔵（run_python を使えるようにする）

パッケージ後、`sys.executable` は Python ではなく `Mochi.exe` になるため、それで直接 `run_python` を走らせると Mochi 自身が起動してしまう。そこで `build.ps1` は追加で**公式の embeddable Python + pip** をダウンロードし、`dist\Mochi\pyruntime\` に配置する。`run_python` / `install_package` はパッケージ版（`frozen` 時のみ）で自動的にこれを使うよう切り替わり、開発時には影響しない。

- pyruntime は**クリーン**（標準ライブラリ + pip のみ）である。サードパーティライブラリ（requests / numpy など）を使うには、まず `install_package` で pyruntime にその場でインストールする必要がある —— 開発版はプリインストール済み、パッケージ版は必要に応じてインストールする、という正常な差異である。
- `build.ps1` の末尾で `pip OK / 欠落` を表示する。欠落していれば `build.ps1` を再実行するか、手動で補う：embeddable Python を `dist\Mochi\pyruntime\` に解凍し、`._pth` を編集して `import site` を有効化し、`get-pip.py` を一度実行する。

### 5.4 初回パッケージでよくある調整（重依存アプリの通弊）

パッケージし終えたら、まず**ダブルクリックで一度走らせ、各機能を一通りクリックして**みて、エラーに応じて `mochi.spec` を補ってください。

- **OCR を使うとすぐクラッシュ / モデルが見つからない** → RapidOCR のモデルが収集しきれていない。spec はすでに `collect_data_files("rapidocr_onnxruntime")` 済みだが、それでも欠ける場合はエラーのパスに従い `datas` で `.onnx` を補う。
- **`ModuleNotFoundError`** → モジュール名を spec の `hiddenimports` に追加する。
- **ウィンドウが起動しない / Qt プラグインが欠落** → spec 冒頭の `collect_all("PySide6")` でカバー、またはコマンドラインで `--collect-all PySide6`。
- **`uiautomation` / `comtypes` のエラー**（画面の閲覧 / コントロールのクリック）→ spec はすでに `comtypes*` を含むが、それでもエラーが出れば `--collect-all comtypes` を追加。
- アイコンの追加：`.ico` のパスを `mochi.spec` の `icon=` に記入する。
- ウイルス対策ソフトが PyInstaller で作った exe を誤検出することがある（通弊。信頼に追加すればよい）。`build\`、`dist\` はすでに `.gitignore` 済み。

## 6. テストとトラブルシューティング

自動化された UI テストはなく、**手動ウォークスルーのチェックリスト**で項目ごとに検証します（操作 → 期待）。実行前に Mochi が起動済みで、コントロールパネルに API が設定済みであることを確認してください。観察する経路は 3 つ。

1. ペットのそばの**吹き出し / 黒板 / ポラロイド / 確認パネル**；
2. **コントロールパネル**（インターフェース / 対話 / 権限 / About の 4 ページ）；
3. **監査ログ** `data/logs/audit-YYYYMMDD.jsonl`（`Get-Content` で最新数行を確認）。

ウォークスルーのカバー範囲：基本的な対話と表情、コマンドアクション（perform / 小芝居）、アイドル行動、割り込み、自発メッセージ、記憶とエピソードログ、グローバルホットキー、クリップボードとウィンドウ、コントロールパネル、リマインドとスケジュール、思考アニメーション、黒板と画像、振り返りのゲート制御。

**よくあるトラブルシューティング**：

- **返答言語が正しくない** → コントロールパネル「対話」ページの「返答言語」欄（空 = あなたが話した言語に追従）。
- **一部の能力が「できない」** → 「権限」ページで対応する能力グループ（ネット接続 / 操作 / コマンド）をオフにすると、ツールがモデルのツール表から隠され、これは期待どおりの挙動。
- **自発メッセージが出ない** → 節制している：在席 + ビジーでない + rapport 到達 + クールダウン経過が必要。素早く検証したければ自発頻度を「おしゃべり」にする。それでも即時にはトリガーされない。
- **起動時に「ホットキーを取得できなかった」と表示** → `Ctrl+Alt+S` が他のソフトに占有されている。ペットをクリックすれば引き続きチャット可能。
- **純粋な雑談の後には振り返りのモデル呼び出しが発生しないはず**（短い雑談は振り返りをスキップ）。ツールを使った実質的なタスクのみが振り返りを行い、記憶 / ログ / 自己像を更新する可能性がある。

> 完全な項目別チェックリストは **[TESTING.md](TESTING.md)** を参照。

## 7. 能力と安全性

Mochi は**あなたのマシン上で任意のコマンドとコードを実行し、マウス・キーボードを操作し、ファイルを読み書きできます** —— これが強力さの源泉であり、同時にリスクも意味します。

- 本質的に**あなた本人と同等の PC 操作権限**を持つ。
- コントロールパネルで能力を**グループ単位でオフ**にできる（ネット接続 / 操作 / コマンド実行）。不安な場面では権限を下げられる。
- **不可逆 / 高リスクな操作は `confirm` パネルを経由する**。「実行 / 中止」をポップアップし、あなたがうなずいて初めて動き出す。
- **安全ガードレール**（`executor/safety.py`）は、**ごく小さく、高精度で、破滅的に不可逆な**操作（ディスクのフォーマット、`diskpart` / `remove-partition`、`reg delete HKLM /f`、生ドライブのルート / システムディレクトリへの再帰的強制削除など）のみをハードに阻止する。あえてサンドボックスは設けず、通常のファイル削除は阻止しない。
- API Key などの設定、記憶 / ナレッジベース / ログはすべて**ローカル保存**。「記憶を消去」ボタンでワンクリック消去できる（自己像を含み、出荷時の地色へ戻る）。
